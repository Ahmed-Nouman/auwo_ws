# Excavation Site Model (Gazebo)

This model includes an **excavation pit** mesh derived from the [MathWorks autonomous-excavator](https://github.com/mathworks-robotics/autonomous-excavator) repo (`ExcavatorCADFiles/Membrane_points.csv`). The pit is generated by the script `scripts/membrane_points_to_pit_stl.py` (scale and depth are configurable). You can replace or add meshes (quarry, soil piles) as described below.

## Option 1: Use the MathWorks-derived pit (default)

The pit mesh `meshes/excavation_pit.stl` is generated from [MathWorks autonomous-excavator](https://github.com/mathworks-robotics/autonomous-excavator) `ExcavatorCADFiles/Membrane_points.csv`. The world includes this model at `(2, 0, 0)`.

To regenerate the pit (e.g. after cloning the MathWorks repo or to change scale/depth):

```bash
cd /path/to/auwo_ws/src/excavator_description
python3 scripts/membrane_points_to_pit_stl.py ../mathworks_autonomous_excavator/ExcavatorCADFiles/Membrane_points.csv -o models/excavation_site/meshes/excavation_pit.stl --scale 150 --depth 2.0
```

## Option 2: Add your own 3D mesh

### Where to get a model

- **Sketchfab** (free, various licenses; check each model):
  - [SenseFly Mining Quarry](https://sketchfab.com/3d-models/sensefly-mining-quarry-3e6c67aa21f949f2b5d80dea2ad33742) – mining quarry
  - [Mining Quarry](https://sketchfab.com/3d-models/mining-quarry-1c86188613284bf8ae3f9f3f85613769) – dense terrain
  - [Quarry](https://sketchfab.com/3d-models/quarry-247a63ac1bbd45ccb0bdd1551a017aad)
  - [Dirty stones pile](https://sketchfab.com/3d-models/dirty-stones-pile-893e64e62f414b439672a8770dd2869d) – small pile
- **Cults 3D**: search “excavation” or “quarry” for STL models.
- **Gazebo Fuel**: [app.gazebosim.org/fuel/models](https://app.gazebosim.org/fuel/models) – search for “construction”, “terrain”, “ground”.

### Formats

Gazebo supports **STL**, **OBJ**, and **DAE** (Collada). Prefer **STL** or **OBJ** for simplicity. If you download **GLB**, convert to STL (e.g. in Blender: File → Export → STL).

### Steps to use your mesh

1. **Put the mesh file here**  
   Copy your file into this folder’s `meshes/` subdirectory:
   ```text
   excavator_description/models/excavation_site/meshes/
   ```
   Example: `meshes/quarry.stl` or `meshes/excavation_site.obj`.

2. **Point the model at your mesh**  
   Edit `model.sdf` in this folder:
   - Remove or comment out the placeholder `<visual>` that uses `<box>`.
   - Uncomment the `<visual name="mesh_visual">` block (the one with `<mesh><uri>model://excavation_site/meshes/...</uri></mesh>`).
   - Set the `<uri>` to your filename, for example:
     - `model://excavation_site/meshes/quarry.stl`
     - `model://excavation_site/meshes/excavation_site.obj`
   - Adjust `<scale>` if the model is too big or small (e.g. `0.5 0.5 0.5` or `2 2 2`).

3. **Rebuild and run**  
   ```bash
   colcon build --packages-select excavator_description excavator_gazebo
   source install/setup.bash
   ros2 launch excavator_interactive_rviz auwo_twin.launch.py
   ```
   Use the excavation world (e.g. `use_excavation_site:=true` if that’s how your launch is set up).

### Position and scale in the world

To move or rotate the excavation site, edit `excavation_site.sdf` in `excavator_description/worlds/` and change the `<include>` pose:

```xml
<include>
  <uri>model://excavation_site</uri>
  <pose>2 0 0 0 0 0</pose>   <!-- x y z roll pitch yaw (rad) -->
  <name>excavation_site</name>
</include>
```

Increase the first number to move it further from the excavator; adjust the last number (yaw) to rotate it.
